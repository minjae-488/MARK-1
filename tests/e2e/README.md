# E2E Tests

이 디렉토리는 End-to-End 테스트 파일을 저장합니다.

## 📋 테스트 시나リ오

Phase 1 MVP의 E2E 테스트는 주로 **수동 테스트**로 진행됩니다.

### 테스트 체크리스트

#### 1. 주문 수집 플로우
- [ ] 스마트스토어에서 테스트 주문 생성
- [ ] Make.com 시나리오 실행 (5분 대기)
- [ ] Airtable Orders 테이블에 주문 추가 확인
- [ ] 주문 정보가 정확한지 검증

#### 2. 합배송 그룹핑 플로우
- [ ] 동일 도매처, 동일 우편번호 앞 3자리인 주문 2개 생성
- [ ] 그룹핑 시나리오 실행
- [ ] Fulfillment_Groups 테이블에 그룹 생성 확인
- [ ] 주문들이 올바른 그룹에 연결되었는지 확인

#### 3. 자동 발주 플로우
- [ ] 그룹의 total_quantity를 5 이상으로 설정
- [ ] 자동 발주 시나리오 실행
- [ ] 도매처에 발주 이메일 발송 확인
- [ ] 그룹 상태가 'ordered'로 변경 확인

#### 4. 송장 등록 플로우
- [ ] 도매처로부터 송장 번호 이메일 수신
- [ ] 송장 매핑 시나리오 실행
- [ ] Orders 테이블에 송장 번호 추가 확인
- [ ] 스마트스토어에 송장 등록 확인
- [ ] 고객에게 알림톡 발송 확인

## 🔧 테스트 도구

### Phase 1 (No-code)
- ✅ Make.com 실행 로그
- ✅ Airtable 테이블 직접 확인
- ✅ 이메일 수신 확인

### Phase 2+ (코드 기반)
- Playwright (E2E 테스트 - UI 제외)
- Supertest (API 테스트)
- Jest (통합 테스트)
