# Product Requirements Document (PRD)
## MARK 1 - AI 기반 자율형 이커머스 통합 솔루션

**문서 버전:** 1.0  
**작성일:** 2026-02-05  
**프로젝트명:** MARK 1 (가칭)  
**핵심 가치:** 데이터 기반 의사결정 자동화 및 운영 리소스 90% 절감

---

## 1. Executive Summary

### 1.1 프로젝트 개요
MARK 1은 중소형 이커머스 셀러를 위한 AI 기반 올인원 운영 자동화 솔루션입니다. 다채널 주문 관리, 재고 최적화, 트렌드 분석, 자동 발주까지 전 과정을 AI가 처리하여 1인 운영자도 대형 쇼핑몰 수준의 물량을 처리할 수 있도록 지원합니다.

### 1.2 비전
"모든 셀러가 운영이 아닌 전략에 집중할 수 있는 세상"

### 1.3 타겟 유저
- **Primary:** 월 매출 1천만원~5천만원 규모의 1인 이커머스 사업자
- **Secondary:** 운영 효율화를 원하는 중소형 온라인 쇼핑몰 (직원 2~5명)
- **Tertiary:** 이커머스 사업 진입을 준비하는 예비 창업자

### 1.4 핵심 문제 정의
현재 중소형 셀러들이 겪는 주요 문제점:
1. **시간 소모:** 다채널 주문 수집 및 처리에 하루 4~6시간 소요
2. **재고 관리 실패:** 과다 재고로 인한 자금 경색 또는 품절로 인한 기회 손실
3. **가격 경쟁력 상실:** 시장 가격 변동 대응 지연으로 역마진 발생
4. **트렌드 대응 지연:** 수동 검색으로 인한 황금 타이밍 놓침
5. **마케팅 비효율:** 개인화되지 않은 일괄 프로모션

### 1.5 솔루션 핵심 가치
- ✅ **운영 시간 90% 절감** (6시간 → 30분)
- ✅ **재고 회전율 150% 향상** (AI 기반 수요 예측)
- ✅ **수익률 평균 15% 개선** (다이내믹 프라이싱)
- ✅ **트렌드 포착 속도 10배 향상** (실시간 크롤링 + AI 분석)

## 2. System Architecture

### 2.1 3-Layer 아키텍처

```
┌─────────────────────────────────────────────────────────┐
│                   ACTION LAYER (실행)                    │
│  Auto-Fulfillment | 송장 맵핑 | 프로모션 자동화         │
└─────────────────────────────────────────────────────────┘
                            ▲
                            │
┌─────────────────────────────────────────────────────────┐
│               INTELLIGENCE LAYER (분석)                  │
│  수요 예측 | 다이내믹 프라이싱 | Creative AI             │
└─────────────────────────────────────────────────────────┘
                            ▲
                            │
┌─────────────────────────────────────────────────────────┐
│                  DATA LAYER (수집)                       │
│  채널 API | 트렌드 크롤러 | 공급망 연동                  │
└─────────────────────────────────────────────────────────┘
```

### 2.2 Layer별 상세 기능

#### 2.2.1 Data Layer (데이터 수집 및 통합)

**A. 다채널 API 커넥터**
- **대상 플랫폼:** 스마트스토어, 쿠팡, 11번가, 지마켓, 옥션
- **수집 데이터:**
  - 주문 정보 (실시간 동기화)
  - CS 문의 및 리뷰
  - 재고 상태
  - 정산 데이터
- **동기화 주기:** 실시간 (Webhook) 또는 5분 간격 polling
- **기술 스택:** REST API, OAuth 2.0, Rate Limiting 처리

**B. 시장 트렌드 크롤러**
- **데이터 소스:**
  - 네이버 데이터랩 (검색량, 연관 키워드)
  - 인스타그램 해시태그 급상승 트렌드
  - 틱톡 인기 상품 태그
  - 경쟁사 판매 순위 (주요 카테고리별)
- **크롤링 주기:** 일 1회 (새벽 2시) + 실시간 이벤트 감지
- **저장 형식:** 시계열 데이터베이스 (InfluxDB)

**C. 공급망 연동**
- **대상 도매처:** 도매매, 이노션, 직거래 업체
- **연동 데이터:**
  - 실시간 재고 수량
  - 단가 변동 정보
  - 배송 가능 일정
- **업데이트 주기:** 1시간 간격
- **예외 처리:** API 장애 시 이메일/카카오톡 알림

#### 2.2.2 Intelligence Layer (AI 분석 및 판단)

**A. 수요 예측 엔진**
- **예측 모델:** LSTM 기반 시계열 분석
- **학습 데이터:**
  - 과거 3개월 판매 데이터
  - 시즌성 패턴 (요일, 월, 분기)
  - 외부 요인 (날씨, 이벤트, 트렌드 급등)
- **출력 결과:**
  - 상품별 7일/14일/30일 예상 판매량
  - 적정 재고 수준 (안전 재고 포함)
  - 품절 임박 알림 (D-3, D-1, D-Day)
- **정확도 목표:** MAPE(평균 절대 백분율 오차) 15% 이하

**B. 다이내믹 프라이싱**
- **가격 결정 로직:**
  ```
  최종 가격 = MAX(
    시장 최저가 × 0.99,
    (도매가 + 배송비) × (1 + 최소마진율)
  )
  ```
- **조정 빈도:** 1일 3회 (오전 9시, 낮 12시, 저녁 6시)
- **안전 장치:**
  - 최소 마진율 설정 (기본 15%)
  - 가격 급락 방지 (1일 최대 -10%)
  - 역마진 자동 경고
- **A/B 테스팅:** 일부 상품 대상 가격 탄력성 실험

**C. Creative AI**
- **기능 1: 상세페이지 카피라이팅**
  - 입력: 상품 카테고리, 주요 키워드, 타겟층
  - 출력: SEO 최적화된 상세 설명 (300~500자)
  - 모델: GPT-4o + 이커머스 특화 Fine-tuning
  - 톤앤매너: 설득력 있는 구매 유도형

- **기능 2: 이미지 생성 (Phase 2)**
  - 입력: 상품 특징, 컨셉 키워드
  - 출력: 상세페이지 배너 이미지, 프로모션 썸네일
  - 모델: DALL-E 3 또는 Midjourney API

#### 2.2.3 Action Layer (자동 실행)

**A. Auto-Fulfillment (자동 주문 처리)**
- **프로세스:**
  1. 다채널 주문 수집 (5분 간격)
  2. 합배송 그룹핑 (같은 도매처 + 배송지 근접)
  3. 재고 확인 및 발주 우선순위 결정
  4. 도매처 API를 통한 자동 발주
  5. 발주 확인 및 예외 사항 알림
- **합배송 로직:**
  - 같은 우편번호 앞 3자리 + 같은 도매처 → 합배송 그룹
  - 배송비 절감 효과 시뮬레이션 제공
- **예외 처리:**
  - 재고 부족 시 → 대체 상품 추천 또는 고객 안내 자동 발송
  - 발주 실패 시 → 관리자 즉시 알림 (카카오톡, 이메일)

**B. 송장 자동 맵핑**
- **프로세스:**
  1. 도매처에서 송장 번호 수신 (API 또는 이메일 파싱)
  2. 주문 번호와 자동 매칭
  3. 각 채널 API를 통해 송장 번호 일괄 등록
  4. 고객에게 배송 시작 알림톡 발송
- **처리 속도:** 송장 수신 후 평균 2분 이내 등록 완료
- **정확도:** 99.5% (오매칭 시 수동 확인 프로세스)

**C. 맞춤형 프로모션**
- **기능 1: 고객 세그먼트별 쿠폰 발행**
  - 휴면 고객 (90일 미구매) → 20% 할인 쿠폰
  - VIP 고객 (월 20만원 이상) → 무료배송 + 포인트 2배
  - 첫 구매 고객 → 재구매 유도 10% 쿠폰
- **기능 2: 재구매 유도 알림톡**
  - 구매 후 30일 → "재구매 추천 상품" 메시지
  - 장바구니 방치 24시간 → "추가 할인" 메시지
- **성과 측정:**
  - 프로모션별 전환율 트래킹
  - ROI 자동 계산 및 리포트

## 3. 핵심 기능 상세

### 3.1 기능 매트릭스

| 카테고리 | 기능명 | 설명 | 우선순위 | MVP 포함 |
|---------|--------|------|---------|---------|
| 운영 자동화 | Zero-Touch 주문 관리 | 수집→발주→송장 등록 자동화 | P0 | ✅ |
| 재고 최적화 | AI 스마트 발주 | 수요 예측 기반 자동 재입고 | P0 | ✅ |
| 트렌드 소싱 | 넥스트 위너 추천 | 황금 키워드 상품 발굴 | P1 | ✅ |
| 마케팅 AI | 개인화 추천 엔진 | 고객별 맞춤 프로모션 | P1 | ⬜ |
| Creative | 상세페이지 자동 생성 | AI 카피라이팅 + 이미지 | P2 | ⬜ |

### 3.2 기능별 상세 스펙

#### 3.2.1 Zero-Touch 주문 관리

**사용자 스토리:**
> "셀러로서, 나는 수동 주문 처리 없이도 모든 채널의 주문이 자동으로 도매처에 발주되고 송장이 등록되기를 원한다. 그래서 운영 시간을 90% 절감하고 전략적 업무에 집중할 수 있다."

**기능 요구사항:**
- [FR-001] 시스템은 스마트스토어, 쿠팡, 11번가의 주문을 5분 간격으로 자동 수집해야 한다.
- [FR-002] 같은 도매처 상품을 합배송 그룹으로 묶어 배송비를 최적화해야 한다.
- [FR-003] 재고 확인 후 도매처 API를 통해 자동 발주해야 한다.
- [FR-004] 발주 실패 시 관리자에게 즉시 알림을 보내야 한다.
- [FR-005] 송장 번호 수신 시 2분 이내에 모든 채널에 자동 등록해야 한다.

**비기능 요구사항:**
- [NFR-001] 주문 처리 정확도 99.9% 이상
- [NFR-002] 시스템 가용성 99.5% 이상 (월 최대 다운타임 3.6시간)
- [NFR-003] 피크 타임(점심, 저녁) 처리 지연 시간 5분 이내

**수용 기준:**
- ✅ 100건의 주문이 인간 개입 없이 자동 처리됨
- ✅ 합배송으로 배송비 평균 30% 절감
- ✅ 송장 등록 오류율 0.5% 이하

#### 3.2.2 AI 스마트 발주

**사용자 스토리:**
> "셀러로서, 나는 AI가 판매 패턴을 학습하여 품절 전에 자동으로 재입고 발주를 하기를 원한다. 그래서 기회 손실과 과다 재고를 모두 방지할 수 있다."

**기능 요구사항:**
- [FR-010] 시스템은 과거 3개월 판매 데이터를 기반으로 7일/14일/30일 판매량을 예측해야 한다.
- [FR-011] 현재 재고가 안전 재고 수준 이하로 떨어지면 자동 발주를 실행해야 한다.
- [FR-012] 품절 예상 3일 전, 1일 전, 당일에 알림을 보내야 한다.
- [FR-013] 시즌성 패턴(요일, 월, 명절 등)을 반영하여 예측 정확도를 높여야 한다.
- [FR-014] 관리자가 안전 재고 수준과 최소 마진율을 설정할 수 있어야 한다.

**비기능 요구사항:**
- [NFR-010] 수요 예측 정확도 MAPE 15% 이하
- [NFR-011] 예측 모델 재학습 주기: 주 1회
- [NFR-012] 예측 결과 생성 시간: 상품당 1초 이내

**수용 기준:**
- ✅ 테스트 기간(1개월) 동안 품절 발생률 5% 이하
- ✅ 재고 회전율 기존 대비 50% 향상
- ✅ 과다 재고(60일 이상 미판매) 30% 감소

#### 3.2.3 넥스트 위너 추천

**사용자 스토리:**
> "셀러로서, 나는 검색량은 높지만 경쟁이 적은 '황금 키워드' 상품을 AI가 자동으로 추천해주기를 원한다. 그래서 블루오션 시장을 선점할 수 있다."

**기능 요구사항:**
- [FR-020] 시스템은 네이버 데이터랩에서 일일 검색량 상위 1,000개 키워드를 수집해야 한다.
- [FR-021] 각 키워드의 경쟁 강도(스마트스토어 상품 수, 평균 리뷰 수)를 계산해야 한다.
- [FR-022] "기회 지수 = 검색량 / (경쟁 상품 수 × 평균 리뷰 수^0.5)" 공식으로 점수를 산출해야 한다.
- [FR-023] 기회 지수 상위 50개 키워드를 주간 리포트로 제공해야 한다.
- [FR-024] 각 키워드에 대한 추천 도매 상품 리스트를 함께 제공해야 한다.

**비기능 요구사항:**
- [NFR-020] 데이터 수집 및 분석 완료 시간: 매일 새벽 4시 이전
- [NFR-021] 추천 정확도: 실제 도입 시 평균 전환율 3% 이상

**수용 기준:**
- ✅ 주간 리포트에 최소 20개 이상의 유효한 키워드 포함
- ✅ 추천 상품의 80%가 도매매에서 구매 가능
- ✅ 베타 사용자 3명 중 2명이 추천 상품으로 실제 판매 시작

## 4. 사용자 여정 (User Journey)

### 4.1 AS-IS (현재 프로세스)

**아침 (2시간)**
1. 스마트스토어 로그인 → 주문 확인 (20분)
2. 쿠팡 로그인 → 주문 확인 (20분)
3. 11번가 로그인 → 주문 확인 (15분)
4. 엑셀로 수동 통합 (30분)
5. 도매처별 수량 계산 (20분)
6. 이메일로 발주 (15분)

**오후 (2시간)**
7. 도매처 송장 수신 확인 (30분)
8. 각 채널에 송장 복사/붙여넣기 (60분)
9. 오류 건 수정 (30분)

**저녁 (2시간)**
10. 네이버 트렌드 검색 (40분)
11. 경쟁사 분석 (40분)
12. 신규 상품 소싱 (40분)

**총 소요 시간: 6시간**

### 4.2 TO-BE (MARK 1 도입 후)

**아침 (10분)**
1. 대시보드 접속
2. AI 생성 [일일 리포트] 확인:
   - 전일 주문 100건 자동 처리 완료
   - 발주 실패 2건 (재고 부족) → 대체 상품 제안
   - 오늘 송장 등록 예정 95건
3. 예외 사항 2건 승인 (클릭 2회)

**점심 (10분)**
4. [주간 트렌드 리포트] 확인:
   - 넥스트 위너 후보 15개
   - 추천 상품 3개 북마크

**저녁 (10분)**
5. [재고 알림] 확인:

## 5. 단계별 구축 로드맵

### 5.1 MVP 전략 (Minimum Viable Product)

**핵심 원칙:**
- "완벽한 전체보다 작동하는 핵심"
- Phase 1에서 가장 큰 Pain Point 해결
- 각 Phase마다 실사용자 피드백 수집

### 5.2 Phase 1: 기반 구축 (4주)

**목표:** 다채널 주문 통합 및 자동 발주의 MVP 완성

**주요 기능:**
- [✅] Airtable 기반 주문 데이터 통합 DB 구축
- [✅] Make.com을 활용한 워크플로우 자동화:
  - 스마트스토어, 쿠팡, 11번가 API 연동
  - 도매매 자동 발주 스크립트
  - 송장 번호 자동 맵핑
- [✅] 관리자 대시보드 (기본):
  - 주문 현황 조회
  - 발주 현황 조회
  - 수동 개입 필요 건 알림

**기술 스택:**
- Backend: Airtable (No-code DB)
- Automation: Make.com (iPaaS)
- Frontend: Airtable Interface or Glide (No-code)
- Notification: 카카오톡 비즈니스 API

**성공 지표:**
- 주문 자동 처리율 80% 이상
- 송장 등록 시간 기존 대비 70% 단축
- 베타 사용자 3명 확보 및 피드백 수집

**Deliverables:**
- 작동하는 자동화 워크플로우
- 관리자 대시보드 (베타)
- 사용 가이드 문서

### 5.3 Phase 2: AI 이식 (6주)

**목표:** OpenAI API 기반 지능형 기능 추가

**주요 기능:**
- [🔲] 상품명 SEO 최적화:
  - 입력: 원본 상품명
  - 출력: 검색 최적화된 제목 (35자 이내)
- [🔲] 상세페이지 자동 생성:
  - 입력: 카테고리, 주요 특징 5개
  - 출력: 설득형 상세 설명 (HTML)
- [🔲] CS 자동 응답 (베타):
  - 자주 묻는 질문 20개 학습
  - 95% 신뢰도 이상일 때만 자동 답변
- [🔲] 주간 트렌드 리포트:
  - 네이버 데이터랩 크롤링
  - 기회 지수 계산 알고리즘
  - 이메일 자동 발송

**기술 스택:**
- AI: OpenAI GPT-4o API
- Web Scraping: Puppeteer + Bright Data
- Data Pipeline: Python + Airflow (or Make.com Advanced)

**성공 지표:**
- AI 생성 카피 사용률 60% 이상
- 트렌드 추천 상품의 실제 등록률 40% 이상
- 카피 품질 만족도 4.0/5.0 이상

**Deliverables:**
- AI 카피라이팅 모듈
- 트렌드 분석 리포트 자동 생성 시스템
- Phase 1 대비 개선된 대시보드

### 5.4 Phase 3: 지능화 (8주)

**목표:** 머신러닝 기반 예측 및 최적화 시스템 구축

**주요 기능:**
- [🔲] 수요 예측 모델:
  - LSTM 기반 시계열 분석
  - 최소 3개월 데이터 학습
  - 7일/14일/30일 판매량 예측
- [🔲] 다이내믹 프라이싱:
  - 실시간 시장 가격 크롤링
  - 마진율 기반 자동 가격 조정
  - A/B 테스트 프레임워크
- [🔲] 개인화 추천 엔진:
  - 고객 구매 이력 분석
  - Collaborative Filtering
  - 연관 상품 추천 + 쿠폰 자동 발행
- [🔲] 재고 최적화:
  - 안전 재고 자동 계산
  - 품절 임박 알림 (D-3, D-1)
  - 자동 재발주 트리거

**기술 스택:**
- ML: TensorFlow/PyTorch + scikit-learn
- Database: PostgreSQL + TimescaleDB (시계열)
- Cache: Redis (실시간 가격 데이터)
- Monitoring: Grafana + Prometheus

**성공 지표:**
- 수요 예측 MAPE 20% 이하
- 품절 발생률 5% 이하
- 재고 회전율 50% 향상
- 다이내믹 프라이싱으로 수익률 10% 개선

**Deliverables:**
- 수요 예측 대시보드
- 자동 가격 조정 시스템
- 개인화 마케팅 자동화 모듈

### 5.5 Phase 4: 확장 (지속)

**목표:** 기능 고도화 및 시장 확장

**계획:**
- 추가 채널 연동 (AliExpress, 아마존 글로벌)
- 모바일 앱 출시 (React Native)
- Multi-tenant SaaS 전환
- 파트너십: 물류사, PG사 연동

**규정 준수:**
- 개인정보보호법 준수 (최소 수집 원칙)
